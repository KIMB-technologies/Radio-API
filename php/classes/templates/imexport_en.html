An export is an export file (*.json). All known radios of this installation of Radio-API are always exported, i.e., for each radio the list of radio stations, podcasts, stations last listened to via RadioBrowser, and podcast episodes already listened to. The list for assigning GUI-Codes to radios is also exported.

You can also automate the export and thus create regular backups. The call to %%SERVERURL%%gui/im-export.php?task=export&token=<token> returns an export file.

Please select an export file (*.json) for the import and enter the token for authentication. You can choose whether you want to append the content from the export during import, overwrite only a single radio or replace everything.
Append
All radios and elements from the export are added to this installation of Radio-API. This means that additional radios will be set up for this server. Make sure that the radios (their Macs) have not yet connected to this server.
Overwrite single radio
Overwrite the contents of a single radio from the export with a single radio in this installation of Radio-API. To do this, enter the GUI-Code of the radio in the export (what you want to import) and the GUI code of the radio in the system/installation (into which you want to import). This will overwrite all data of one radio in this installation of Radio-API.
Replace everything
Replace the contents of this installation of Radio-API completely with the contents of the export. All data will be overwritten!
Finally, start the import by clicking the Start import button; incorrect fields are highlighted in red.
